⏱️ Synchronous vs Asynchronous

🔴 Synchronous (متزامن)

الكود بينفذ سطر سطر، وماينفعش سطر يشتغل غير لما اللي قبله يخلص.

تشبيه: واقف في طابور، محدش بيتحرك غير لما اللي قبله يمشي.

🟢 Asynchronous (غير متزامن)

الكود يقدر يطلب حاجة وتاخد وقت، ويكمل شغله عادي لحد ما الحاجة دي تيجي.

تشبيه: طلبت أكل، وسيبت رقمك، ورجعت لما يرن 📞

🔴 كابوس الـ Blocking (التعطيل)

يعني إيه Blocking؟

لما يكون في سطر كود بياخد وقت طويل (زي Loop ضخم)، المعالج بيفضل مشغول فيه ومش عارف ينفذ أي سطر تاني، ولا حتى الـ UI يقدر يستجيب لضغطة المستخدم!

"شوفت يا باشمهندس! الـ For Loop دي خلت الـ PRINT اللي بعدها تستنى كتير جداً.. طيب ما بالك لو السطر ده كان بيحمل بيانات من النت؟"
// مثال على الـ Blocking (التعطيل)
print('Start');

for (int i = 0; i < 1000000000; i++) {
  // كود بياخد وقت طوووويل جداً
}

print('End'); // السطر ده هيفضل محبوس!

🕒 فهم الـ Future & Future.delayed

يعني إيه Future؟

هو كائن (Object) بيمثل قيمة أو خطأ لسه "هيحصل" في المستقبل. زي كأنك طلبت أوردر دليفري، اللي معاك دلوقتي هو "وصل" (Future) بالأوردر اللي لسه هيوصلك.

ليه Future.delayed؟

بنستخدمها عشان نقول للغة: "من فضلك استني كذا ثانية قبل ما تنفذي الكود ده، بس من غير ما توقفي باقي البرنامج". ده جوهر الـ Non-blocking!

void main() {
  print('1. بداية البرنامج');

  // حجز مهمة للمستقبل (3 ثواني)
  Future.delayed(Duration(seconds: 3), () {
    print('2. تم تنفيذ المهمة المؤجلة ✅');
  });

  print('3. تم تنفيذ السطر ده فوراً (بدون انتظار!) 🚀');
}

(بعد 3 ثواني)

2. تم تنفيذ المهمة المؤجلة

توضيح بالصور: ليه المحمول محتاج Async؟

تفاصيل مهمة:

إحنا بنحتاج الـ Async عشان الموبايل فيه معالج (Process) واحد شغال على الـ UI الرئيسي. لو عطلناه، المستخدم هيحس إن التطبيق "تجمد" (App Not Responding).

  • تحميل الصور الكبيرة من السيرفر.
  • جلب بيانات من ملفات ضخمة.
  • الربط مع الـ API الخارجي.

👨‍🍳 قصة المطعم: الـ Async و الـ Await بالكود

1. الـ async (المترجم الذكي)

لما بنكتب async قبل الـ Function، إحنا بنقول لـ Dart: "جهزي نفسك، الوراكي دي هتاخد وقت، وممكن ترجع Future". ده معناه إن الوظيفة دي "غير متزامنة".

2. الـ await (الانتظار الذكي)

كلمة await بنستخدمها جوه الـ async بس. وظيفتها إنها بتمسك الـ "Future" وتقوله: "أنا هستناك هنا لحد ما تخلص وترجع داتا، بس مش هعطل باقي البرنامج اللي بره الـ Function دي".

"يا سلام! يعني الـ await زي الويتر اللي بيستنى المطبخ يخلص، بس المطعم نفسه شغال وباقي الزبائن بتطلب عادي؟"
// مثال واقعي لطلب بيتزا
Future<void> orderPizza() async {
  print('1. طلبت البيتزا من الويتر 🍕');

  // الويتر بينتظر المطبخ (3 ثواني)
  String result = await preparePizza();

  print('3. الويتر جابلي: $result');
}

Future<String> preparePizza() async {
  await Future.delayed(Duration(seconds: 3));
  return 'بيتزا سخنة! 🔥';
}

void main() {
  orderPizza();
  print('2. أنا بقرا كتاب عقبال ما البيتزا تجهز 📖');
}
تسلسل التنفيذ:
  1. - بتطلب البيتزا (1. طلبت البيتزا..)
  2. - المطبخ بيبدأ يشتغل..
  3. - البرنامج يكمل (2. أنا بقرا كتاب..)
  4. - بعد 3 ثواني (3. الويتر جابلي البيتزا!)

⚠️ التعامل مع الأخطاء (Try / Catch)

كارثة بدون Try/Catch:

تخيل لو السيستم بيقسم رقمين، والزبون كتب الرقم التاني "صفر". السيستم هيقع فوراً!

int x = 10 ~/ 0; // ❌ CRASH!!!

الحل مع Try/Catch:

إحنا بنقول له: "جرب" (Try) تنفذ الكود ده، ولو حصلت "قفشة" (Catch)، اتعامل معاها بلطف بدل ما تقفل التطبيق.

// كود آمن (Robust)
try {
  int result = 10 ~/ 0;
  print(result);
} catch (e) {
  print('عفواً، لا يمكن القسمة على صفر!');
  print('نوع الخطأ: $e');
} finally {
  print('تمت المحاولة بنجاح.');
}

🗄️ ما هي قواعد البيانات وأنواعها؟

التعريف المبسط

⚖️ مقارنة شاملة: REST vs SOAP

🚀 RESTful API (المستقبل)

  • خفيف جداً وسريع في نقل البيانات.
  • بيستخدم JSON (بسيط وسهل القراءة).
  • المفضل تماماً لمطوري الـ Mobile و Web.
// مثال لشكل JSON
{
  "name": "Mohamed",
  "age": 30
}

🐢 SOAP API (الماضي)

  • تقيل جداً بسبب تعقيد الهيكل.
  • بيستخدم XML (مليان وسوم صعبة).
  • نادر الاستخدام حالياً إلا في بنوك قديمة جداً.
<!-- مثال لشكل XML -->
<user>
  <name>Mohamed</name>
  <age>30</age>
</user>
ليه JSON أفضل؟

JSON أسهل للقراءة بالنسبة للإنسان، وأسرع للتحويل (Parsing) بالنسبة للكمبيوتر، وحجمه أصغر بكتير في النقل عبر الشبكة مقارنة بـ XML اللي بيضيع مساحة في كتابة الـ Tags.

📦 محرك القوة: pub.dev و الـ Packages

ما هو الـ Package؟

هو كود كتبه مبرمجين تانيين عشان يحلوا مشكلة معينة (زي التعامل مع النت)، وأنت بتاخد الكود ده جاهز وتستخدمه في مشروعك عشان متخترعش العجلة من جديد.

ما هو pub.dev؟

هو "المتجر" أو المستودع الرسمي لكل الـ Packages الخاصة بـ Dart و Flutter. أي حاجة محتاجها (خرايط، كاميرا، API) هتلاقيها هناك.

ليه بنحتاجه؟

  • دعم من مجتمع المبرمجين.

📄 ملف الـ pubspec.yaml

ماهو هذا الملف؟

هو "البطاقة الشخصية" لمشروعك. فيه بنكتب اسم المشروع، وصفه، والأهم من كده: المكتبات (Dependencies) اللي المشروع محتاجها عشان يشتغل.

ليه بنستخدمه؟

  • إضافة مكتبات خارجية (Packages).
  • تعريف الصور والخطوط (Assets).
  • تحديد إصدار الـ SDK المناسب.

🔄 الـ Request والـ Response

1. الـ Request (الطلب)

هو الرسالة اللي الموبايل بيبعتها للسيرفر، وفيها أنواع (Methods) بتحدد إحنا عاوزين إيه:

  • GET لطلب بيانات (زي عرض المنتجات)
  • POST لإرسال بيانات (زي إضافة مستخدم)
  • PUT لتعديل بيانات موجودة بالكامل
  • DELETE لمسح بيانات من السيرفر

2. الـ Response و الـ Status Codes

هو رد السيرفر عليك، وأهم حاجة فيه هي الـ Status Code اللي بيقولك الحالة:

200
OK / Success ✅ الطلب تم بنجاح والداتا وصلت.
201
Created ✨ تم إنشاء الداتا اللي بعتها بنجاح.
404
Not Found ❌ الرابط اللي بتطلبه غلط أو مش موجود.
500
Internal Error 🔥 المشكلة من السيرفر نفسه مش من عندك.

🛠️ خطوات التعامل مع الـ API في الكود بالتفصيل

1. إضافة المكتبة (Dependency)

بنحتاج مكتبة خارجية اسمها http لأن Flutter الأساسي مش بيكلم سيرفرات لوحده. بتكتبها في ملف pubspec.yaml.

dependencies:
  http: ^0.13.5 # دي المكتبة اللي بتمكنا نتكلم مع السيرفر

2. تحويل الرابط (Uri.parse)

السيرفر بيفهم حاجة اسمها Uri مش مجرد نص (String)، فلازم نحول اللينك اللي معانا لشكل الـ Uri عشان نستخدمه.

3. التأكد من النجاح (statusCode == 200)

قبل ما نستخدم الداتا، دايماً بنكشف على الـ statusCode. لو هو 200 يبقى تمام وكمل، غير كده يبقى فيه غلط.

4. فك الشفرة (jsonDecode)

الداتا بتيجي من النت كـ "نص طويل" (String) والكمبيوتر مش فاهمه. بنستخدم jsonDecode عشان نحوله لـ Map أو List حقيقية.

// مثال كامل وشرح مفصل
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<void> getData() async {
  // 1. نجهز الرابط ونحوله بـ Uri.parse
  var url = Uri.parse('https://api.example.com/users');

  // 2. نبعت الطلب ونستنى الرد (await)
  var response = await http.get(url);

  // 3. نكشف على الـ Status Code قبل ما نفك الداتا
  if (response.statusCode == 200) {
    // 4. نحول النص لـ داتا حقيقية بـ jsonDecode
    var data = jsonDecode(response.body);
    print(data[0]['name']); // اطبع أول اسم جالك
  } else {
    print('فيه مشكلة: ${response.statusCode}');
  }
}
"يا محمد، الـ statusCode هو أهم فحص تعمله عشان البرنامج ميخرجش فجأة لو السيرفر وقع!"

📡 مثال حي مع API حقيقي

شوفت السهولة؟

في الكود اللي في الشمال، إحنا بنكلم سيرفر JSONPlaceholder وده سيرفر مشهور المبرمجين بيستخدموه للتجارب. الطلب ده بيجيب بيانات "بوست" واحد بس من السوشيال ميديا.

"يا هندسة، يعني السطر بتاع الـ Uri.parse ده هو اللي بيحدد أنا رايح أجيب أنهي داتا بالظبط؟"
import 'package:http/http.dart' as http;
import 'dart:convert';

Future<void> fetchPost() async {
  final url = Uri.parse('https://jsonplaceholder.typicode.com/posts/1');
  final response = await http.get(url);

  if (response.statusCode == 200) {
    var post = jsonDecode(response.body);
    print('العنوان: ${post['title']}');
    print('المحتوى: ${post['body']}');
  }
}

🛡️ كود احترافي (API + Try Catch)

ليه الـ Try Catch ضروري هنا؟

تخيل لو الزبون فتح البرنامج وهو "قافل النت". لو مش مستخدمين try/catch، البرنامج هيقفل (Crash) في وش المستخدم. الـ catch بتخلينا نعرفه إن فيه مشكلة في الشبكة بلطف.

  • - SocketException: لو مفيش نت خالص.
  • - FormatException: لو السيرفر بعت داتا مش JSON.
Future<void> fetchSafe() async {
  try {
    final response = await http.get(
      Uri.parse('https://api.com/post/1')
    );

    if (response.statusCode == 200) {
      print(jsonDecode(response.body));
    } else {
      print('فشل الطلب: ${response.statusCode}');
    }
  } catch (error) {
    print('خطأ غير متوقع (تأكد من النت): $error');
  }
}

  • الـ API هو لسان الحوار.

🚀 الخطوة الجاية:

  • تحويل البيانات المعقدة لموديل Class.
  • بناء واجهة مستخدم حقيقية بالداتا من النت.
  • التعامل مع الصور والملفات الكبيرة.

📝 التحدي العملي الشامل

التمرين 1: Extension Functions

ابحث عن مفهوم Extension Functions في Dart وقم بتنفيذ المهام التالية:

  • لـ List: أنشئ دالة تمديد (Extension) تقوم بقلب ترتيب العناصر.
  • لـ int: أنشئ دالة تحول الرقم لسلسلة نصية مع إضافة فواصل للآلاف (مثال: 1000 يصبح 1,000).
  • لـ DateTime: أنشئ دالة تعيد التاريخ بصيغة "يوم/شهر/سنة".

التمرين 2: التعامل مع Products API

سنستخدم مكتبة http للاتصال بـ API حقيقي: dummyjson.com/products

  • 1. جلب الكل: أنشئ دالة تجلب قائمة بجميع المنتجات وتعرضها في التطبيق.
  • 2. جلب بالهوية (ID): دالة تستقبل ID المنتج من المستخدم وتجلب بيانات هذا المنتج فقط (/products/2).
  • 3. تحديد العدد (Limit): دالة تستقبل رقم وتجلب عدد محدد من المنتجات (/products?limit=5).
  • 4. البحث الذكي: دالة تستقبل كلمة بحث وتجلب النتائج الموافقة لها (/products/search?q=phone).
  • 5. الأمان والسرعة: التعامل مع حالة الخطأ (Error Handling) وإظهار رسالة "جاري التحميل..." أثناء الجلب.
