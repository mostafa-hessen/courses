الحل الذكي: الـ Custom Widget ✅

coffee_card.dart (Modular) 
class CoffeeCard extends StatelessWidget {
  final String title;
  final String price;

  CoffeeCard({required this.title, required this.price});

  @override
  Widget build(BuildContext context) {
    return Container(
      padding: EdgeInsets.all(20),
      margin: EdgeInsets.symmetric(vertical: 8, horizontal: 16),
      decoration: BoxDecoration(
        color: Colors.white,
        borderRadius: BorderRadius.circular(20),
        boxShadow: [BoxShadow(color: Colors.black12, blurRadius: 10)],
      ),
      child: Row(children: [
        Icon(Icons.coffee_maker, color: Colors.brown),
        SizedBox(width: 15),
        Text(title, style: TextStyle(fontWeight: FontWeight.bold)),
      ]),
    );
  }
}

قبل أن تلمس الكيبورد:

  • 1. تقسيم الشاشة: هل التصميم رأسي (Column) أم أفقي (Row)؟
  • 2. المكونات المكررة: أي عنصر تراه مرتين هو مرشح ليكون Custom Widget.
  • 3. التمرير (Scroll): هل العناصر ستخرج خارج الشاشة؟ إذاً نحتاج لـ ListView.

الفرق بين الـ Padding والـ Margin

Margin (فراغ يبعد العنصر عن جيرانه)
Padding (حشو يبعد المحتوى عن الحواف)
محتوى (Text / Icon)

Padding

يخلق مساحة "داخلية". الصندوق يكبر من الداخل.

Margin

يخلق مساحة "خارجية". الصندوق يدفع من حوله.

كيف يفكر مبرمج فلاتر كـ "مهندس"؟ 🧠

1. الرؤية الشاملة

النظر للتصميم ككل وتحديد الألوان والخطوط المشتركة (Theme).

2. تقسيم المكونات

أي زر أو صندوق مكرر يتم استخراجه كـ Custom Widget فوراً.

3. إدارة البيانات

من أين ستأتي أسعار القهوة؟ وما هي الأشكال التي سنعرضها؟

"لا تبدأ في الكود حتى تفهم خريطة الطريق"

كابوس وضع كل الكود في main.dart 😱

main.dart (1,500 lines...) 
// ... 300 line of imports ...
void main() => runApp(MyApp());
class MyApp extends StatelessWidget {
  // 500 lines of Widget tree
  // Column -> Container -> Row -> Column -> Container...
  // Repeated styles everywhere...
  // Repeated Colors.brown everywhere...
  // IF YOU WANT TO CHANGE ONE THING... YOU ARE DEAD!
}

لماذا هذا الأسلوب فاشل؟

  • ❌ استحالة التعديل: تغيير بسيط قد يخرب التطبيق كله.
  • ❌ فشل التعاون: لا يستطيع مبرمجان العمل على نفس الملف.
  • ❌ صعوبة التتبع: البحث عن خطأ في 1000 سطر مستحيل.
  • ❌ التكرار الممل: كتابة نفس التصميم لكل صنف قهوة.

"التنظيم هو أول خطوة نحو العالمية"

التنظيم المتوسط للمشاريع الكبيرة

lib

models(هذا ملف البيانات)

ما هو الـ Models؟

هو القلب الذي يحمل بيانات القهوة (الاسم، السعر، الصورة) بشكل منظم يفهمه الكود.

ما هو الـ Services؟

هو المكان الذي نكتب فيه الأكواد التي تجلب البيانات من الإنترنت أو قاعدة البيانات.

التنظيم الاحترافي (Modular Architecture) 👑

1. Assets Folder

  • 📁 assets/fonts
  • 📁 assets/images
  • 📁 assets/icons

2. lib / core

  • 📁 constant (الألوان والكلمات الثابتة)
  • 📁 theme (ألوان التطبيق الموحدة)
  • 📁 utils (أدوات مساعدة)

3. lib / features

  • 📁 home:

    • data: المسؤول عن جلب البيانات (APIs) والـ Models الخاصة بالرئيسية.

    • widgets: العناصر الصغيرة التي لا تستخدم إلا في شاشة الـ Home.

    • view: الشاشات الكاملة (Screens) التي يراها المستخدم.

  • 📁 auth (تسجيل الدخول)
  • 📁 orders
  • كل ميزة مستقلة بملفاتها لمنع تداخل الكود.

هذا التنظيم يجعلك "Senior" في نظر الشركات الكبرى.

كيف ندرج الصور في المشروع؟ 🖼️

الخطوة الأولى: المجلد

ننشئ مجلد اسمه `assets` وبداخله `images` ونضع الصور فيه.

الخطوة الثانية: pubspec.yaml 

flutter:
  assets:
    - assets/images/logo.png
    - assets/images/    # تفعيل المجلد كله

الخطوة الثالثة: الكود 

Image.asset(
  'assets/images/logo.png',
  width: 200,
  height: 200,
)

أناقة التطبيق: Google Fonts ✍️

إذا أعجبك خط على Google Fonts، فإضافته أسهل مما تتخيل!

1. حمل ملف الخط (OTF/TTF).
2. ضعه في assets/fonts.
3. عرفه في pubspec.yaml.
pubspec.yaml Fonts 
flutter:
  fonts:
    - family: MyFont
      fonts:
        - asset: assets/fonts/Tajawal-Bold.ttf
          weight: 700
