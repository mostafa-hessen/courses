المتغيرات = "درج المكتب"

المتغير (Variable) هو مكان في ذاكرة الكمبيوتر (درج)، له اسم، وله نوع معين بس يقدر يشيله.
مينفعش تحط 'مكواة' في درج 'الأقلام'!

درج النصوص (String) String product = "Milk";
درج الأعداد (int) int stock = 15;
درج الكسور (double) double weight = 1.5;
درج المنطق (bool) bool hasOffer = false;

Null Safety: درع الحماية

نظام بيمنعك تستخدم "درج فاضي" وأنت مش واخد بالك، عشان البرنامج ميضربش في وش العميل!

التحكم (Control Flow)

إشارات المرور في الكود (If, Else, Switch). بتقول للكود يمشي يمين ولا شمال.

الثابت والمتغير (var vs const)

var: درج ممكن نغير محتواه.
const: لحام أمير! اللي يتحط ميتغيرش.

الثوابت: Final vs Const

Final (ثابت بس مرن)

"بكتبه مرة واحدة بس لما أعرفه، حتى لو وقت التشغيل".
ينفع ياخد قيمة من دالة أو وقت ما البرنامج شغال.

final name = "Ali"; // تمام
final time = DateTime.now(); // تمام (وقت التشغيل)
// name = "Omar"; // خطأ! مينفعش تغيره

Const (محفور من المصنع)

"ثابت وقيمته معروفة والبرنامج لسه بيكتب (Compile Time)".
مينفعش ياخد قيمة متغيرة أو وقت التشغيل.

const pi = 3.14; // تمام
// const time = DateTime.now(); // خطأ!
"أنا بكتب الأسماء في النوتة، بس ساعات الزبون بيمشي من غير ما يقول اسمه، وساعات بيقولي رقم تليفونه! أعمل إيه؟"

الإدخال وفزورة Null Safety

فزورة الـ Null Safety

?
علامة الشك (Nullable)

"يا عم محمد، العميل ممكن ميردش". المتغير يقبل قيمة أو يبقى فاضي (null).

String? name;
!
علامة الثقة العمياء (Bang Operator)

"أنا حالف إنه هيرد!". بجبر الكمبيوتر يتعامل مع القيمة على أنها موجودة. خطر: لو مردش البرنامج هيفرقع!

String realName = name!;

استقبال رقم من العميل Challenge

import 'dart:io';

print("دخل رقمك يا بيه:");
// 1. نستقبل النص الأول
String? input = stdin.readLineSync();
// 2. نحوله لرقم (Parsing)
int number = int.parse(input!);
print("الرقم بعد التحويل: $number");

معلومة خطيرة للـ VS Code!

عشان الـ Input يشتغل معاك في الـ Terminal، لازم تعدل ملف launch.json:

"console": "terminal"

الحلقات التكرارية (Loops): التكرار بذكاء

1. For Loop

لما أكون عارف هعد كام مرة بالظبط.

مثال واقعي:

عم محمد بيعبي 50 زجاجة عصير.

for (int i=1; i<=50; i++) {
  fillBottle(i);
}

2. While Loop

طالما الشرط موجود، شغال (مش عارف العدد).

مثال واقعي:

فضل ياكل قصب لحد ما بطنه وجعته.

while (isHungry) {
  eatCane();
}

3. Do-While Loop

نفذ مرة واحدة الأول، وبعدين شوف الشرط.

مثال واقعي:

دوق العصير الأول، لو عجبك اشرب تاني.

do {
  tasteJuice();
} while (isTasty);

محاكاة خط الإنتاج

اضغط تشغيل للبدء...

التحكم في التكرار: مشاكل عم محمد

Continue

المشكلة: عم محمد لقى برتقالة بايظة في القفص.

الحل: يرميها ويكمل عد باقي البرتقال (مش هيقفل الدكان عشان برتقالة!).

for (var i=1; i<=10; i++) {
  if (orange == "rotten") {
    print("فوتها!");
    continue;
  }
  count();
}

Break

المشكلة: عم محمد لقى ثعبان 🐍 في القفص!

الحل: يسيب القفص ويجري فوراً (يوقف العد تماماً).

while (true) {
  if (foundSnake) {
    print("يا لهوي!");
    break;
  }
  count++;
}

الدوال (Functions): مبدأ DRY

Don't Repeat Yourself (لا تكرر نفسك)

ليه بنستخدم دوال؟

تشريح الدالة (Function Anatomy)

void ReturnType
makeTea Name
(int sugar) Parameters
{ ... } Body
  • تنظيف الكود:

    بدل ما الكود يبقى "سلطة"، بنقسمه لمكعبات صغيرة كل مكعب بيعمل وظيفة وحدة.

  • إعادة الاستخدام:

    كتبت كود "عمل الشاي" مرة؟ تقدر تستدعيه 1000 مرة من غير ما تكتب الخطوات تاني.

  • سهولة الصيانة:

    لو اكتشفنا إن السكر ناقص، بنعدل في مكان واحد بس (الدالة) مش في كل البرنامج.

Before (WET Code)

تكرار ممل (Write Everything Twice)

print("سخن المية");
print("حط الشاي");
print("قلب");
// عميل تاني جه
print("سخن المية");
print("حط الشاي");
print("قلب");
After (DRY Code)

نظافة (Don't Repeat Yourself)

void makeTea() {
  print("سخن، حط، قلب");
}
makeTea(); // أول عميل
makeTea(); // تاني عميل

أمثلة ومشاكل: ليه بنحتاج دوال؟

المشكلة

عم محمد كل شوية يحسب سعر العصير لزبون مختلف. بيكتب نفس الكود كل مرة!

// زبون 1
int price1 = 10 * 2; // مانجو
print("Hese is your check: $price1");

// زبون 2
int price2 = 5 * 3; // قصب
print("Hese is your check: $price2");

الحل (Function)

نعمل دالة ونناديها بس!

void printCheck(int price, int qty) {
  int total = price * qty;
  print("Hese is your check: $total");
}

// الاستخدام
printCheck(10, 2);
printCheck(5, 3);

الفرق القاتل: Print vs Return

Print (الويتر المنادي)

الويتر بيصرخ ويقول "2 شاي وصلوا!". هو بس بيعرّف الناس (بيعرض ع الشاشة)، لكن مش بيديك الشاي في إيدك.

void sum(a, b) {
  print(a + b);
}
var x = sum(5,10); // خطأ! x هتكون null

Return (الشيف اللي بيسلمك)

الشيف بيطبخ الأكل وبيديهولك في إيدك (Return). تقدر تاخده تاكله، تديه لحد تاني، أو تحطه في الثلاجة (Variable).

int sum(a, b) {
  return a + b;
}
var x = sum(5,10); // صح! x بقيمة 15

معاملات الدالة (Parameters)

يعني إيه Parameters؟

دي المدخلات اللي الماكينة محتاجاها عشان تشتغل.
تخيل "خلاط العصائر" (Function). عشان يطلع عصير، محتاج "فاكهة" و "مية" و "سكر" (Parameters).
من غيرهم الخلاط هيلف ع الفاضي!

1. Positional (بالترتيب)

الترتيب مهم جداً! الأول للأول.

void juice(String fruit, int sugar) { ... }
// لازم فاكهة الأول ثم سكر
juice("Mango", 3);

2. Named (بالاسم) 🌟

الأكثر استخداماً في Flutter. الترتيب مش مهم.

void juice({required String fruit, int? sugar})
// براحتي في الترتيب
juice(sugar: 2, fruit: "Mango");

3. Default Value (الافتراضي)

لو الزبون مقالش، حطله من عندك.

void juice({String type = "Cane"})
juice(); // هيطلع قصب لوحده

4. Optional Positional

اختياري بس لو حطيته لازم بترتيبه.

void juice(String fruit, [bool? ice])
juice("Lemon"); // من غير تلج عادي

أنواع الدوال (Function Types)

1. Arrow Function (Lambda)

المختصرة. لو الدالة بتعمل حاجة واحدة بس في سطر واحد.

// الطريقة العادية
int sum(a, b) {
  return a + b;
}

// Lambda (Arrow)
int sum(a, b) => a + b;

2. Anonymous Function

الدالة المجهولة (من غير اسم). بتستخدم مرة واحدة زي عند الضغط على زر.

ElevatedButton(
  onPressed: () {
    print("Button Clicked!");
  },
  child: Text("Click Me"),
);
button.addEventListener("click", function() {
  console.log("Button Clicked!");
});
// Or ES6 Arrow
button.onclick = () => console.log("Hi");

مثال حقيقي: دكان عم محمد

عم محمد عايز برنامج يحسب "سعر الطلب" ويضيف "الضريبة" ويعمل "خصم" لو الزبون حبيبه.

double calculateOrder({
  required double price,
  required int quantity,
  bool isVip = false,
}) {
  double total = price * quantity;

  // خصم الحبايب 10%
  if (isVip) {
    total = total * 0.9;
  }

  // الضريبة 14%
  total = total * 1.14;

  return total;
}

زبون عادي:

calculateOrder(price: 10, quantity: 2); -> 22.8

زبون VIP (خصم):

calculateOrder(price: 10, quantity: 2, isVip: true); -> 20.52

Higher Order Functions

دي دوال "باشا"، بتاخد دوال تانية تشغلها عندها، أو بترجع دالة كنيجة.
زي "المقاول" اللي بيجيب "نجار" و "سباك" (دوال صغيرة) يخلصوا الشغل.

سحر التعامل مع البيانات (Map, Where)

Map (التحويل)

بتمشي على عنصر عنصر وتحوله لحاجة تانية.

[1, 2, 3]
[2, 4, 6]
var nums = [1, 2, 3];
var doubled = nums.map((n) => n * 2);

Where (الفلترة)

بتنقي العناصر اللي بتحقق شرط معين بس.

[1, 5, 10]
[5, 10]
(اكبر من 4)
var nums = [1, 5, 10];
var big = nums.where((n) => n > 4);

القوائم (Lists)

"قائمة الطلبات بتاعتي كبرت! ورقة وقلم واكتبي ورايا يا ست الكل..."

أهم نوع لتخزين مجموعة بيانات مع بعض

List<String> menu = [
  "قصب",
  "سوبيا",
  "خروب"
];

print(menu[0]); // قصب

العمليات الأساسية

  • list.add("Apple") // إضافة للنهاية
  • list.insert(0, "Apple") // إضافة في الأول
  • list.remove("Apple") // حذف بالقيمة
  • list.removeAt(2) // حذف بالترتيب

خصائص مهمة

list.length

عدد العناصر في القائمة.

list.isEmpty

هل القائمة فاضية ولا فيها حاجة؟

list.first / list.last

أول وآخر عنصر سريعاً.

مكتبة الرياضيات (dart:math) 🧮

عم محمد بيحسب مكاسبه ومصاريفه، محتاج أدوات قوية.

import 'dart:math';

print(max(10, 5)); // -> 10
print(min(10, 5)); // -> 5
print(sqrt(9)); // -> 3.0
print(Random().nextInt(10)); // -> 0..9

🔥 مثال: رسالة تحفيزية عشوائية

var quotes = ["عاش", "كمل", "وحش"];
var i = Random().nextInt(quotes.length);
print(quotes[i]);

💪 تحديات الأبطال

1. الآلة الحاسبة

اعمل دالة calc(a, b, op) بتاخد رقمين وعلامة (+, -, *, /) وترجع النتيجة.

2. الأسعار (List + Map)

عندك قائمة أسعار [10, 20, 30]. استخدم .map عشان تزود ضريبة 14% على كل سعر.

3. كلمة السر (Logic)

دالة بتتشك على الـ Password: لازم تكون أطول من 6 حروف ومش بتيأدي بمسافة.

4. البحث (Where)

قائمة أسماء ["Ali", "Amr", "Samy"]. فلتر ورجع الأسماء اللي بتبدأ بحرف "A" بس.

5. المضروب (Factorial)

اكتب دالة تحسب مضروب الرقم. مثلاً factorial(5) ترجع 120 (اللي هي 5*4*3*2*1).

6. الباليندروم (Palindrome)

اكتب دالة تتأكد لو الكلمة بتتقري زي بعض من اليمين والشمال (زي "radar" أو "level").

7. العدد الأولي (Prime)

اكتب دالة بتاخد رقم وترجع true لو هو أولي (مبيقبلش القسمة غير على نفسه والواحد).

8. أكبر رقم (Max)

اكتب دالة بتاخد قائمة أرقام [10, 5, 20, 3] وترجع أكبر رقم فيهم يدوي (من غير ما تستخدم مكتبة Math).
