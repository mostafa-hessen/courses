مراجعة مفاهيم MUI: sx و variant

في MUI، يمكننا تخصيص الشكل بسهولة باستخدام خاصيتين أساسيتين:

  • build variant: شكل جاهز من الثيم مثل: contained, outlined, text
  • style sx: أسلوب كتابة CSS داخل الـ theme system باستخدام JS object
  • edit يمكن الجمع بين sx و variant لنفس المكون
<Button variant="contained" sx={{ bgcolor: 'primary.main', color: '#fff' }} />

جدول مقارنة

الخاصية الغرض مثال
variant تطبيق شكل جاهز variant="outlined"
sx تخصيص دقيق للستايل sx={{ color: 'primary.main' }}

أهم مكونات MUI التي استخدمناها

  • text_fields Typography – لعرض النصوص مع variant
  • smart_button Button – مع variant و color
  • view_stream Stack – لترتيب العناصر عموديًا أو أفقيًا
  • grid_view Grid – لتقسيم الواجهة واستجابة الشاشة
  • account_circle Avatar – لصورة المستخدم
  • credit_card Card – لعرض محتوى منسق مثل البوستات
  • apps Icon – لاستخدام أيقونات من مكتبة MUI

ملاحظات سريعة:

  • معظم هذه المكونات تدعم `sx` لتخصيص التصميم
  • يمكن دمجها بسهولة لبناء واجهة تطبيق كاملة
  • تعتمد على theme افتراضي ويمكن تخصيصه

أسئلة مراجعة شفوية

  • ما الفرق بين sx و style؟ وأيهما يُفضل؟ ولماذا؟
  • ما وظيفة variant في المكونات؟ واذكر مثالًا عمليًا.
  • ما الفرق بين Stack و Grid؟ ومتى تستخدم كلًا منهما؟
  • اذكر 3 مكونات رئيسية من MUI استخدمناها، ومتى تستخدم كل واحدة.
  • كيف نضيف أيقونة بجانب زر؟ مع مثال سريع.

مقدمة إلى MUI Theme

Theme في توثيق MUI

الTheme هو نظام تصميم متكامل في MUI يسمح لك بتحديد مظهر التطبيق من حيث الألوان، المسافات، الحدود، الخطوط والمزيد. يتم تطبيق الـ Theme تلقائيًا على جميع المكونات لضمان التناسق.

ما يحتويه الـ Theme الافتراضي

  • palette: ألوان المكونات (primary, secondary, error...) 🎨 نظام الألوان في MUI (Color Palette)
  • typography: أنماط الخطوط (fontSize, fontFamily, headings)
  • spacing: وحدات المسافات (1 = 8px)
  • shape: إعدادات الشكل مثل borderRadius
  • breakpoints: نقاط التحول للشاشات المختلفة
  • components: تخصيص مكونات معينة (مثل: Button, Card...)
  • zIndex: ترتيب الطبقات مثل AppBar و Modal
  • transitions: مدة وتأثيرات الحركة

ماذا يحدث عند كتابة <Button color="primary">؟

  • يتم تمرير الـ prop color="primary" إلى مكون Button
  • داخل مكتبة MUI، يتم تحويل هذا إلى كلاس CSS يعتمد على اللون: مثل .MuiButton-containedPrimary
  • الكلاس يحتوي على:
    • background-color: theme.palette.primary.main
    • color: theme.palette.primary.contrastText
    • عند Hover: background-color: theme.palette.primary.dark
  • الكود المسؤول عن ذلك داخل ملف: Button.js
ملاحظة: color="primary" تعمل فقط كـ prop داخل المكونات، لكن لو أردت استخدام اللون في خصائص مثل sx أو style مثل:
sx={{ bgcolor: 'primary' }} ❌ لن تعمل
بل يجب كتابة: sx={{ bgcolor: 'primary.main' }} ✅

مثال عملي:

<Button 
  variant="contained" 
  color="primary"
>
  Click Me
</Button>

هذا الزر سيأخذ اللون theme.palette.primary.main كخلفية، والنص سيكون بلون contrastText.

⚙️ الخصائص الأساسية في Theme في MUI

يحتوي Theme الافتراضي في MUI على مجموعة من الخصائص القابلة للتخصيص والتي تتحكم في مظهر التطبيق بالكامل. إليك أهم هذه الخصائص مع القيم الافتراضية:

🎨 palette - نظام الألوان

  • primary.main: #1976d2
  • secondary.main: #9c27b0
  • error.main: #d32f2f
  • warning.main: #ed6c02
  • info.main: #0288d1
  • success.main: #2e7d32

🔤 typography - إعدادات الخط

  • fontFamily: "Roboto", "Helvetica", "Arial", sans-serif
  • fontSize: 14px
  • h1: 96px
  • h2: 60px
  • h3: 48px

📐 shape, spacing, zIndex

  • spacing: 1 = 8px
  • shape.borderRadius: 4px
  • zIndex.modal: 1300
  • zIndex.appBar: 1100

📱 breakpoints - نقاط التحول

  • xs: 0
  • sm: 600px
  • md: 900px
  • lg: 1200px
  • xl: 1536px

🧪 أمثلة واقعية على استخدام الخصائص

🎨 مثال استخدام palette: 
sx={{ bgcolor: 'primary.main', color: 'primary.contrastText' }}
🔤 مثال استخدام typography: 
<Typography variant="h3">مرحبا بكم في MUI</Typography>
📐 مثال على spacing: 
sx={{ p: 2, m: 3 }}  // padding: 16px, margin: 24px
🧱 مثال استخدام shape: 
<Box sx={{ borderRadius: 2 }} />  // 16px (2 * 8)
📱 مثال على breakpoints: 
<Box sx={{ fontSize: { xs: 12, sm: 14, md: 16 } }} />
// يتغير الحجم حسب حجم الشاشة

🛠️ تخصيص الـ Theme باستخدام createTheme

يتيح لك MUI تعديل التصميم العام للتطبيق عن طريق إنشاء Theme مخصص وتغليف التطبيق به باستخدام ThemeProvider.

مثال على تخصيص الألوان

import { createTheme, ThemeProvider } from '@mui/material/styles';

const theme = createTheme({
  palette: {
    primary: {
      main: '#1976d2',
      light: '#63a4ff',
      dark: '#004ba0',
      contrastText: '#fff'
    },
    secondary: {
      main: '#9c27b0'
    }
  },
});

<ThemeProvider theme={theme}>
  <App />
</ThemeProvider>

👈 باستخدام createTheme يمكنك تعديل الألوان، الخطوط، الهوامش، الزوايا وغيرها.

👈 يتم تمرير الكائن إلى ThemeProvider لتطبيقه على جميع المكونات.

👈 يمكنك دمج أكثر من تعديل في نفس الوقت: palette + typography + components

👈 عند تخصيص مكون معين (مثل الزر)، يمكن استخدام components داخل الثيم لتعديل الأنماط الافتراضية.

✨ هذا يسمح بتوحيد تصميم الواجهة بسهولة والتحكم الكامل بالمظهر.

تخصيص الثيم - نظام الألوان (Palette)

نظام الألوان هو أحد أهم أجزاء الثيم حيث يتحكم في الألوان الأساسية لتطبيقك.

خصائص نظام الألوان

الخاصية الوصف مثال
primary.main اللون الأساسي للمكونات
primary.light نسخة فاتحة من اللون الأساسي
primary.dark نسخة داكنة من اللون الأساسي
secondary.main اللون الثانوي للمكونات
error.main لون لحالات الخطأ
warning.main لون للتحذيرات
success.main لون لحالات النجاح

const theme = createTheme({

    secondary: {

      light: '#ff9e40',
      dark: '#c43e00',
    },

      main: '#d32f2f',

  },

تخصيص الثيم - الطباعة (Typography)

نظام الطباعة يتحكم في كل ما يتعلق بالخطوط في تطبيقك، من العناوين إلى النصوص العادية.

خصائص نظام الطباعة

عنوان رئيسي (h1)

عنوان ثانوي (h2)

عنوان فرعي (h3)

نص عادي (body1)

نص صغير (body2)

تخصيص الطباعة 

const theme = createTheme({
  typography: {
    fontFamily: [
      'Tajawal',
      '-apple-system',
      'BlinkMacSystemFont',
      '"Segoe UI"',
      'Roboto',
      '"Helvetica Neue"',
      'Arial',
      'sans-serif',
    ].join(','),
    h1: {
      fontSize: '2.5rem',
      fontWeight: 700,
      lineHeight: 1.2,
    },
    h2: {
      fontSize: '2rem',
      fontWeight: 600,
      lineHeight: 1.3,
    },
    body1: {
      fontSize: '1rem',
      lineHeight: 1.6,
    },
    button: {
      textTransform: 'none',
      fontWeight: 600,
    },
  },
});

نصائح لتخصيص الطباعة:

  • اختر خطًا يدعم اللغة العربية جيدًا مثل Tajawal أو Cairo
  • تأكد من أن أحجام الخطوط مناسبة للقراءة على جميع الشاشات
  • استخدم ألوانًا مناسبة للنصوص حسب خلفيتها
  • احترم التدرج الهرمي للنصوص (عناوين، نصوص فرعية، نصوص عادية)

🧩 إضافة مفاتيح مخصصة داخل Theme في MUI

يمكنك إضافة أي مفتاح مخصص داخل الثيم، مثل mostafa، لاستخدامه في تطبيقك يدويًا.

💡 لماذا نستخدم مفاتيح مخصصة؟

  • لتخزين ألوان أو قيم ثابتة خاصة بتصميمك.
  • لتوحيد استخدام الأنماط الخاصة بفريقك أو مشروعك.
  • للاستفادة منها عند استخدام useTheme().

❗ ملاحظات مهمة:

  • لن تؤثر المفاتيح المخصصة على مكونات MUI تلقائيًا.
  • يجب استخدامها يدويًا عبر كود مثل theme.mostafa.text.

مثال عملي

const theme = createTheme({
  palette: {
    primary: { main: '#1976d2' }
  },
  mostafa: {
    text: '#fff',
    accent: '#ff4081'
  }
});
        
const theme = useTheme();

<Typography sx={{ color: theme.mostafa.text }}>
  نص باستخدام اللون المخصص
</Typography>
✅ يمكنك الآن استخدام القيم المخصصة في أي مكان داخل التطبيق عبر useTheme().
✨ تعتبر هذه الطريقة مفيدة جدًا لتنظيم التصميم المخصص داخل مشروع كبير أو متعدد الصفحات.

🛠️ تخصيص الـ Theme باستخدام createTheme

يتيح لك MUI تعديل التصميم العام للتطبيق عن طريق إنشاء Theme مخصص وتغليف التطبيق به باستخدام ThemeProvider.

مثال كامل على تخصيص الثيم


import { createTheme, ThemeProvider } from '@mui/material/styles';
import { Button, Typography, Box, Paper } from '@mui/material';

const theme = createTheme({
  palette: {
    primary: {
      main: '#4a148c',
      light: '#7c43bd',
      dark: '#12005e',
    },
    secondary: {
      main: '#ff6f00',
    },
  },
  typography: {
    fontFamily: 'Tajawal, sans-serif',
    fontSize: 16,
    h1: {
      fontSize: '2.5rem',
      fontWeight: 700,
    },
  },
  shape: {
    borderRadius: 12,
  },
  spacing: 8,
});

export default function App() {
  return (
    <ThemeProvider theme={theme}>
      <Box sx={{ p: 4 }}>
        <Typography variant="h1" gutterBottom>تجربة الخط</Typography>
        <Button variant="contained" color="primary">زر رئيسي</Button>
        <Button variant="outlined" color="secondary" sx={{ ml: 2 }}>زر ثانوي</Button>

        <Paper elevation={3} sx={{ mt: 4, p: 2, borderRadius: 3 }}>
          <Typography variant="body1">محتوى داخل ورقة Paper</Typography>
        </Paper>

        <Box sx={{ mt: 4, p: 2, bgcolor: 'primary.light', borderRadius: 2 }}>
          <Typography variant="body2" color="primary.contrastText">
            مربع بخلفية primary.light وحواف مدورة
          </Typography>
        </Box>
      </Box>
    </ThemeProvider>
  );
}

👈 يتم تمرير الكائن إلى ThemeProvider لتطبيقه على جميع المكونات.

👈 يمكنك دمج أكثر من تعديل في نفس الوقت: palette + typography + components

👈 المثال يحتوي على:

  • زر رئيسي وثانوي بالألوان المخصصة
  • نص بعنصر Typography يستخدم الخط المعدل
  • عنصر Paper بحواف دائرية
  • عنصر Box مع خلفية primary.light
✨ هذا يسمح بتوحيد تصميم الواجهة بسهولة والتحكم الكامل بالمظهر.

واجهة برمجة التطبيقات المنسقة (styled API)

ما هي واجهة styled API؟

واجهة styled API في MUI تسمح لك بإنشاء مكونات مخصصة باستخدام CSS-in-JS مع دعم كامل للثيم.

مميزات استخدام styled API:

  • تكامل كامل مع نظام الثيم
  • كتابة أنماط مخصصة للمكونات
  • استخدام قيم الثيم في الأنماط
  • دعم للوسائط المتعددة (Responsive Design)
  • أداء أفضل من CSS التقليدي في بعض الحالات

إنشاء مكون مخصص 

import { styled } from '@mui/material/styles';
import Button from '@mui/material/Button';

// إنشاء زر مخصص باستخدام styled API
const CustomButton = styled(Button)(({ theme }) => ({
  padding: theme.spacing(2),
  borderRadius: theme.shape.borderRadius * 2,
  fontWeight: theme.typography.fontWeightBold,
  boxShadow: theme.shadows[4],
  '&:hover': {
    boxShadow: theme.shadows[8],
    backgroundColor: theme.palette.primary.dark,
  },
  [theme.breakpoints.down('sm')]: {
    fontSize: theme.typography.body2.fontSize,
  },
}));

// استخدام المكون المخصص
<CustomButton variant="contained">
  زر مخصص
</CustomButton>

🌙 الوضع الداكن (Dark)

راحة للعين ليلاً

👨‍💻 مثال على تحديد الوضع داخل createTheme

import { createTheme } from '@mui/material/styles';

const theme = createTheme({
  palette: {
    mode: 'dark', // يمكنك تغييره إلى 'light'
  },

🌙 مثال عملي متقدم للوضع الليلي والنهاري

هذا المثال يعرض كيفية تطبيق الوضع الليلي والنهاري باستخدام createTheme مع ThemeProvider و CssBaseline وتغيير palette.mode ديناميكيًا.

كود المثال:

<ThemeProvider theme={{theme}}>
  <CssBaseline />
  <AppBar position="static" color="primary">
    <Toolbar>
      <Typography variant="h6" sx={{ flexGrow: 1 }}>
        MUI Dark & Light Mode
      </Typography>
      <Brightness7 />
      <Switch
        checked={{mode === 'dark'}}
        onChange={{() => setMode(mode === 'light' ? 'dark' : 'light')}}
        color="default"
      />
      <Brightness4 />
    </Toolbar>
  </AppBar>

  <Box sx={{ p: 4 }}>
    <Typography variant="h4" gutterBottom>
      الوضع الحالي: {mode}
    </Typography>

    <Grid container spacing={{4}}>
      <Grid item xs={{12}}>
        <Paper sx={{ p: 3 }}>
          <Typography variant="h6" gutterBottom>
            نموذج تسجيل الدخول
          </Typography>
          <TextField fullWidth label="البريد الإلكتروني" margin="normal" />
          <TextField fullWidth label="كلمة المرور" type="password" margin="normal" />
          <Button variant="contained" color="primary" fullWidth sx={{ mt: 2 }}>
            تسجيل الدخول
          </Button>
        </Paper>
      </Grid>

      <Grid item xs={{12}}>
        <Card>
          <CardContent>
            <Typography variant="h6">معلومات المستخدم</Typography>
            <Typography color="text.secondary">الاسم: مصطفى حسين</Typography>
            <Typography color="text.secondary">الدور: مطور واجهات أمامية</Typography>
          </CardContent>
        </Card>

        <Box sx={{ mt: 3, p: 2, bgcolor: 'secondary.main', borderRadius: 2 }}>
          <Typography color="secondary.contrastText">
            هذا صندوق بلون ثانوي ويتغير مع الوضع الداكن.
          </Typography>
        </Box>
      </Grid>
    </Grid>
  </Box>
</ThemeProvider>

🌗 الوضع الليلي والنهاري يتم تحديده عبر الخاصية palette.mode.

🌓 يتم تمرير الوضع داخل createTheme ليتغير تصميم التطبيق بناءً على القيمة: 'light' أو 'dark'.

🧩 مكون CssBaseline يستخدم لضبط الأنماط الافتراضية للمتصفح لتتناسب مع الثيم وتوحيد الشكل بين المتصفحات.

✨ عند استخدام ThemeProvider نغلف التطبيق أو جزء منه لتطبيق الثيم عليه.

🔁 يمكنك استخدام زر تبديل أو Switch لتغيير الوضع بشكل ديناميكي.

CssBaseline يقوم بإزالة الفروقات الافتراضية بين المتصفحات ويوفر قاعدة تصميم نظيفة وموحدة لجميع المكونات.
