ما هي مكتبة MUI؟

MUI (المعروفة سابقًا باسم Material-UI) هي مكتبة React UI framework مبنية على مبادئ Material Design من جوجل. الإصدار 7 هو أحدث إصدار يحوي تحسينات كبيرة.

Material Design هو نظام تصميم أنشأته شركة Google لتوحيد وتسهيل تصميم واجهات المستخدم عبر الويب والتطبيقات. يرتكز على مبادئ ويهدف لتقديم تجربة استخدام واضحة، متناسقة، وسهلة التفاعل.

مقارنة بين MUI و Bootstrap

المعيار MUI Bootstrap
النمط التصميمي Material Design من جوجل تصميم خاص (أكثر عمومية)
المكونات مخصصة لـ React عامة (تعمل مع أي إطار عمل)
التخصيص عن طريق الثيم (Theme) أو الـ Props عن طريق CSS أو SASS
سهولة التعلم متوسطة (تتطلب معرفة React) سهلة (تعمل بمعرفة HTML/CSS فقط)
حجم الحزمة ~300KB (مع الأيقونات) ~200KB

التثبيت والإعداد في مكتبة MUI

📦 أمر التثبيت الأساسي 

npm install @mui/material @emotion/react @emotion/styled

🎨 تثبيت الأيقونات 

npm install @mui/icons-material

1. @mui/material

مكتبة المكونات الجاهزة مثل: Button, AppBar, Card.

import Button from '@mui/material/Button';

2. @emotion/react

المحرك الأساسي لأنماط CSS-in-JS داخل MUI. يستخدم لتطبيق التنسيقات باستخدام خاصية sx.

<Box sx={{ padding: 2 }} />

3. @emotion/styled

تُستخدم لإنشاء مكونات مخصصة بأنماط مرنة باستخدام `styled`.

const Box = styled('div')({ backgroundColor: 'red' });
⚠️ ملاحظة: مكتبة MUI تعتمد بشكل كامل على Emotion لتطبيق الأنماط منذ الإصدار الخامس.

ما هو الـ Component في React؟

Component هو وحدة بناء مستقلة في واجهة المستخدم. يُمكنك التفكير فيه كـ "قطعة ليغو" تُستخدم لإنشاء أجزاء مختلفة من التطبيق.

✨ لماذا نستخدم الـ Components؟

  • إعادة الاستخدام: اكتب الكود مرة وكرر استخدامه بسهولة.
  • تنظيم المشروع: يسهل تقسيم التطبيق إلى أجزاء صغيرة.
  • المرونة: يمكن تمرير بيانات إليه عبر الـ Props.

❌ بدون استخدام Component: 

function App() {
  return (
    <div>
      <h1>أهلاً بك في موقعي</h1>
      <p>هذا نص تعريفي</p>
    </div>
  );
}

✅ باستخدام Component و Props: 

function Header(props) {
  return (
    <div>
      <h1>مرحبًا {props.name} 👋</h1>
      <p>هذا نص مخصص لك</p>
    </div>
  );
}

function App() {
  return (
    <div>
      <Header name="Mostafa" />
    </div>
  );
}

✅ Props تسمح لنا بتمرير معلومات إلى المكون من الخارج، تمامًا كما نفعل مع المكونات الجاهزة في MUI مثل Button و TextField.

🔗 كيف ترتبط الفكرة بمكتبة MUI؟

مكتبة Material UI (MUI) تعتمد على تقديم Components جاهزة مثل الأزرار، الحقول، البطاقات وغيرها، لتساعدك على بناء واجهات جذابة وسريعة بدون الحاجة لكتابة CSS معقد.

🎯 مثال على زر من MUI: 

import Button from '@mui/material/Button';

function App() {
  return (
    <Button variant="contained" color="primary">
      اضغط هنا
    </Button>
  );
}

في هذا المثال، نمرر خصائص (props) إلى زر MUI مثل: variant="contained" و color="primary". هذه الخصائص هي التي تجعل الزر يظهر بشكل مختلف، تمامًا كما نفعل مع المكونات التي نكتبها نحن.

✅ من خلال تعلم Components و Props، ستكون جاهزًا للعمل مع MUI بكل احترافية وسرعة!

كيف يعمل زر MUI خلف الكواليس؟ 🎯

🔹 الكود المستخدم: 

import Button from '@mui/material/Button';

function App() {
  return (
    <Button variant="contained" color="primary">
      اضغط هنا
    </Button>
  );
}

🧠 ماذا يحدث خلف الكواليس؟

  1. يتم تمرير variant و color إلى المكون.
  2. يقوم الزر بتحليل الـ props وتحديد الشكل:
    • هل الزر مملوء؟ (contained)
    • ما لونه؟ (primary, secondary...)
    • هل الزر معطل؟
  3. يتم توليد classNames مثل MuiButton-containedPrimary.
  4. يستخدم مكون داخلي اسمه ButtonBase.
  5. يتم تطبيق التصميم بناءً على إعدادات theme المركزي: الثيم هو الذي يحدد الألوان، الخطوط، والمسافات الافتراضية.
  6. النتيجة النهائية: زر HTML احترافي جاهز للعرض.

🧾 النتيجة النهائية في الـ DOM: 

<button class="MuiButton-root MuiButton-contained MuiButton-containedPrimary">
  اضغط هنا
</button>

🧩 ما هو الـ Theme في MUI؟

الثيم هو النظام الذي يتحكم في الألوان، الخطوط، الزوايا، المسافات، وغير ذلك، ويضمن توحيد التصميم في كل المشروع. الزر في MUI يعتمد على الـ theme لتحديد لونه الأساسي، حجم الخط، والظل.

🔗 اقرأ المزيد عن الـ Theme من الصفحة الرسمية: mui.com/material-ui/customization/theming

✅ زر MUI هو مكون React ذكي، يتفاعل مع Props، ويُنسّق تلقائيًا باستخدام Theme جاهز واحترافي.

كيف كُتب زر MUI في الكود الحقيقي؟ 💻

لما تكتب الكود التالي:

<Button variant="contained" color="primary">
  اضغط هنا
</Button>

فعليًا إنت بتستخدم مكون React مكتوب بنفسك، ولكن بطريقة احترافية داخل MUI. لنشوف بعض الأجزاء من كود الزر الأصلي الموجود في GitHub.

👀 اضغط هنا لعرض الكود الأصلي على GitHub

🧠 ماذا يحدث داخل ملف Button.js؟

  • المكون مكتوب باستخدام React.forwardRef لدعم تمرير المراجع (ref).
  • يستقبل كل الخصائص (props) التي تكتبها، مثل variant و color.
  • يستخدم دالة اسمها useThemeProps لدمج props مع إعدادات الثيم.
  • يتم إنشاء متغير اسمه ownerState يحتوي كل props.
  • يتم تحديد الـ class النهائية باستخدام useUtilityClasses(ownerState).
  • الزر الحقيقي يتم إنشاؤه باستخدام مكون داخلي اسمه ButtonRoot.

مثال مبسط من داخل الملف: 

const Button = React.forwardRef(function Button(inProps, ref) {
  const props = useThemeProps({ props: inProps, name: 'MuiButton' });
  const {
    children,
    variant = 'text',
    color = 'primary',
    ...other
  } = props;

  return (
    <ButtonRoot className="..." ref={ref} ...>
      {children}
    </ButtonRoot>
  );
});

✅ في النهاية: زر MUI هو مجرد مكون React بيستقبل props، وبيستخدمها في صناعة زر HTML شكله احترافي باستخدام theme و classes.

مكون Typography في MUI

text_fields

نظرة عامة

مكون Typography يُستخدم لعرض النصوص وفق نظام تصميم Material Design، ويوفر 13 تنسيقًا جاهزًا مثل العناوين والنصوص العادية.

أهم الخصائص:

  • variant: تحديد نوع النص (مثل h1, h2, body1...)
  • align: محاذاة النص (left, center, right...)
  • color: لون النص (primary, secondary, error...)
  • gutterBottom: إضافة هامش سفلي تلقائي
ℹ️ معلومة: هناك فرق بين component (المكون نفسه مثل Typography) و component API (الصفحة التي تشرح خصائصه وكيفية استخدامه بالتفصيل).
📚 اقرأ المزيد عن خصائص Typography من الصفحة الرسمية: mui.com/material-ui/api/typography

مثال على الاستخدام 

import Typography from '@mui/material/Typography';

<Typography variant="h4" component="h1" gutterBottom>
  عنوان رئيسي
</Typography>

<Typography variant="body1" color="text.secondary">
  نص عادي مع لون ثانوي
</Typography>

ما هي خاصية sx في MUI؟ 🎨

خاصية sx هي طريقة ذكية تقدمها MUI لتطبيق الستايلات مباشرة داخل JSX باستخدام JavaScript Object. وهي جزء من MUI System الذي يحتوي على خصائص جاهزة مثل: m / p / color / display وغيرها.

✅ مميزات استخدام sx:

  • تكتب CSS كـ JavaScript Object.
  • دعم كامل للـ Theme مثل: 'primary.main'.
  • يدعم Pseudo Selectors مثل &:hover.
  • يدعم Responsiveness مباشرة.
<Box
  sx={{
    bgcolor: 'primary.main',
    p: 2,
    '&:hover': { bgcolor: 'primary.dark' },
    color: 'white',
    borderRadius: 2,
    textAlign: 'center',
  }}
>
  محتوى داخل Box
</Box>
🧠 MUI System يوفر خصائص جاهزة يمكن استخدامها داخل sx مباشرة.
مثال: بدل padding: "16px"، نكتب p: 2.
🔗 قائمة الخصائص: mui.com/system/properties

🤔 props أم sx؟

  • استخدم props الجاهزة إن كانت متوفرة (مثل color="primary").
  • استخدم sx للتخصيص الكامل والمعقد.
  • يمكن الدمج بين الاثنين بحرية.
⚙️ من الذي يشغّل sx؟
مكتبة @emotion/react هي المسؤولة عن تحويل كود sx إلى CSS فعلي داخل الصفحة.
عند استخدام MUI، يتم تثبيت Emotion تلقائيًا لأنها المحرّك الأساسي لكتابة الستايلات في المكتبة.

✅ sx = طريقة MUI السحرية لكتابة CSS مرن + معتمد على Theme + مدعوم من Emotion.

📝 نموذج تسجيل دخول باستخدام MUI

هذا مثال عملي على استخدام مكون TextField و Button من MUI لبناء نموذج بسيط يحتوي على معالجة حالة الأخطاء. 


import { useState } from 'react';
import { TextField, Button, Box } from '@mui/material';

function LoginForm() {
  const [username, setUsername] = useState('');
  const [password, setPassword] = useState('');
  const [error, setError] = useState(false);

  const handleSubmit = (e) => {
    e.preventDefault();
    if (password.length < 8) {
      setError(true);
    } else {
      setError(false);
      alert('تم تسجيل الدخول!');
    }
  };

  return (
    <Box component="form" onSubmit={handleSubmit} sx={{ maxWidth: 400, mx: 'auto' }}>
      <TextField
        label="اسم المستخدم"
        variant="outlined"
        fullWidth
        margin="normal"
        value={username}
        onChange={(e) => setUsername(e.target.value)}
      />

      <TextField
        label="كلمة المرور"
        type="password"
        variant="outlined"
        fullWidth
        margin="normal"
        error={error}
        helperText={error ? "كلمة المرور يجب أن تكون 8 أحرف على الأقل" : ""}
        value={password}
        onChange={(e) => setPassword(e.target.value)}
      />

      <Button
        type="submit"
        variant="contained"
        color="primary"
        fullWidth
        sx={{ mt: 2 }}
      >
        تسجيل الدخول
      </Button>
    </Box>
  );
}

export default LoginForm;

مكون Menu في MUI

menu

نظرة عامة

قائمة منبثقة تحتوي على قائمة من الخيارات. يتم تنشيطها عادةً عند النقر على زر.

خصائص مهمة:

  • anchorEl: العنصر الذي يتم تثبيت القائمة به
  • open: حالة فتح/إغلاق القائمة
  • onClose: دالة عند إغلاق القائمة
  • anchorOrigin: نقطة تثبيت القائمة
  • transformOrigin: نقطة تحويل القائمة
الخيار الأول
الخيار الثاني
الخيار الثالث

مثال على الاستخدام 

          
import Menu from '@mui/material/Menu';
import MenuItem from '@mui/material/MenuItem';

const [anchorEl, setAnchorEl] = useState(null);

const handleClick = (event) => {
  setAnchorEl(event.currentTarget);
};

const handleClose = () => {
  setAnchorEl(null);
};

<Button onClick={handleClick}>فتح القائمة</Button>
<Menu
  anchorEl={anchorEl}
  open={Boolean(anchorEl)}
  onClose={handleClose}
>
  <MenuItem onClick={handleClose}>الخيار الأول</MenuItem>
  <MenuItem onClick={handleClose}>الخيار الثاني</MenuItem>
</Menu>

مكون Container في MUI

view_compact

نظرة عامة

حاوية مركزية تحدد العرض الأقصى للمحتوى حسب حجم الشاشة. يساعد في تنسيق المحتوى على أجهزة مختلفة.

خصائص مهمة:

  • maxWidth: العرض الأقصى (xs, sm, md, lg, xl, false)
  • fixed: حاوية بعرض ثابت على الشاشات الكبيرة
  • disableGutters: إزالة الهوامش الجانبية
  • يدعم جميع أحجام الشاشات بشكل متجاوب

مثال على الاستخدام 

          
import Container from '@mui/material/Container';

<Container maxWidth="md">
  <!-- محتوى التطبيق هنا -->
</Container>

<Container fixed>
  <!-- محتوى بعرض ثابت -->
</Container>

نظرة عامة

الثيم في MUI

palette

الثيم الافتراضي

يوفر نظامًا متكاملًا للألوان، الطباعة، المسافات، والتكبير/التصغير.

ميزات الثيم الافتراضي:

  • ألوان Material Design الأساسية
  • أنماط Typography محددة مسبقًا
  • مسافات (Spacing) بقيمة 8px كأساس
  • تكبير/تصغير (Breakpoints) متجاوب
  • ظلال (Shadows) محددة مسبقًا

تخصيص الثيم

          
import { createTheme } from '@mui/material/styles';

const customTheme = createTheme({
  palette: {
    primary: { main: '#4a148c' },
    secondary: { main: '#ff6f00' },
  },
  typography: {
    fontFamily: 'Tajawal, Arial',
    h1: { fontSize: '3rem', fontWeight: 700 },
  },
  direction: 'rtl', // للدعم العربي
});

// استخدام الثيم المخصص
<ThemeProvider theme={customTheme}>
  <App />
</ThemeProvider>
