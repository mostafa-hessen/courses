لماذا TypeScript؟

TypeScript تضيف أنواع بيانات ثابتة لـ JavaScript مما يجعل كودك أكثر أماناً وسهولة في الصيانة. في هذه المحاضرة سنتعلم كيفية استخدامها بشكل عملي.

ما هي TypeScript؟

تعريف شامل بأهمية TypeScript

تعريف TypeScript

TypeScript هي لغة برمجة مفتوحة المصدر طورتها مايكروسوفت، وهي مجموعة شاملة (superset) من JavaScript تضيف إليها أنواع بيانات ثابتة (static types).

معلومات أساسية:

  • تاريخ الإصدار: 2012 (الإصدار الأول)
  • المطور: أندرس هايلسبرج (مصمم C#)
  • الشركة: مايكروسوفت
  • الترجمة: يتم تحويل TS إلى JS قبل التنفيذ

أين تستخدم TypeScript؟

  • تطوير تطبيقات الويب (Frontend)
  • تطوير تطبيقات الخادوم (Node.js)
  • تطوير تطبيقات الهاتف (React Native, Ionic)
  • تطوير الألعاب (Unity عبر WebGL)

لماذا نستخدم TypeScript؟

  • اكتشاف الأخطاء أثناء التطوير بدلاً من التنفيذ
  • تحسين تجربة التطوير (Intellisense, Auto-completion)
  • تحسين التعاون في الفرق الكبيرة
  • توثيق الكود تلقائياً عبر الأنواع

عيوب JavaScript

  • Type Coercion المفاجئ (1 + "1" = "11")
  • undefined و null في كل مكان
  • أخطاء وقت التشغيل (Runtime Errors)
  • صعوبة صيانة المشاريع الكبيرة

هل كل ملف JS يعتبر TS؟

نعم، أي ملف JavaScript صالح هو أيضًا ملف TypeScript صالح لأن TypeScript هي مجموعة شاملة لـ JavaScript. يمكنك تغيير امتداد الملف من .js إلى .ts وسيعمل بدون مشاكل.

عيوب JavaScript

وكيف تحل TypeScript هذه المشاكل

المشاكل الشائعة في JavaScript

1. Type Coercion المفاجئ

JavaScript 
// جمع عدد مع نص
const result = 1 + "1";
console.log(result); // الناتج: "11" (غير متوقع)

// مقارنة غير صارمة
const x = 0;
const y = "";
if (x == y) { // true (غير متوقع)
    console.log("متساويان!");
}

المشكلة:

JavaScript تقوم بتحويل الأنواع تلقائياً مما يؤدي إلى نتائج غير متوقعة.

2. أخطاء وقت التشغيل

JavaScript 
// دالة تتوقع عدد ولكن تتلقى نص
function calculateTax(amount) {
    return amount * 0.15;
}

// لن تظهر مشكلة حتى وقت التشغيل
const tax = calculateTax("100"); 
console.log(tax); // "NaN" (غير متوقع)

المشكلة:

لا تكتشف JavaScript الأخطاء المتعلقة بالأنواع إلا أثناء التشغيل.

كيف تحل TypeScript هذه المشاكل؟

1. التحقق من الأنواع أثناء التطوير

TypeScript 
// TypeScript تمنع هذه العمليات
const result: number = 1 + "1"; // Error

const x: number = 0;
const y: string = "";
if (x == y) { // Error
    console.log("متساويان!");
}

الحل:

TypeScript تكتشف هذه الأخطاء أثناء كتابة الكود وليس أثناء التشغيل.

2. تحديد أنواع المعاملات

TypeScript 
// تحديد نوع المعاملات والقيمة المرجعة
function calculateTax(amount: number): number {
    return amount * 0.15;
}

// Error: لا يمكن تمرير نص لدالة تتوقع عدد
const tax = calculateTax("100");

الحل:

TypeScript تفرض تحديد أنواع المعاملات والقيم المرجعة للدوال.

كيف يعمل هذا؟

TypeScript تقوم بتحليل الكود أثناء التطوير (compile-time) وليس أثناء التشغيل (runtime). عند كتابة الكود، تقوم بفحص الأنواع وإظهار الأخطاء فوراً قبل تنفيذ الكود.

Static vs Dynamic Typing

الفرق بين اللغات ذات الأنواع الثابتة والمتحركة

Loosely vs Strongly Typed

Loosely Typed (JavaScript)

  • لا يوجد تحقق من الأنواع أثناء الترجمة
  • التحويل التلقائي بين الأنواع (Type Coercion)
  • أكثر مرونة ولكن أقل أماناً
  • أخطاء تظهر فقط أثناء التشغيل

Strongly Typed (TypeScript)

  • تحقق من الأنواع أثناء الترجمة
  • لا يوجد تحويل تلقائي بين الأنواع غير المتوافقة
  • أكثر صرامة ولكن أكثر أماناً
  • اكتشاف الأخطاء أثناء التطوير

Static vs Dynamic Typing

Statically Typed (Java, C#, TypeScript)

  • يتم تحديد الأنواع أثناء الترجمة
  • اكتشاف الأخطاء قبل التشغيل
  • أداء أفضل في بعض الحالات
  • أكثر ملاءمة للمشاريع الكبيرة

Dynamically Typed (JavaScript, Python, Ruby)

  • يتم تحديد الأنواع أثناء التشغيل
  • أكثر مرونة وسهولة في التعلم
  • أخطاء تظهر فقط أثناء التشغيل
  • أكثر ملاءمة للنماذج الأولية السريعة

Static Typing (TypeScript, Java, C#)

مثال في TypeScript

TypeScript 
// تحديد النوع عند التعريف
let age: number = 25;

// خطأ أثناء التطوير (ليس التشغيل)
age = "خمسة وعشرون"; // Error

// الدوال تحدد أنواع المعاملات والإرجاع
function greet(name: string): string {
    return "مرحباً " + name;
}

greet(42); // Error

مثال في Java

Java 
// Java مثال على
public class Main {
    public static void main(String[] args) {
        int age = 25;
        age = "خمسة وعشرون"; // Error
        
        String message = greet("أحمد");
        System.out.println(message);
    }
    
    public static String greet(String name) {
        return "مرحباً " + name;
    }
}

مميزات Static Typing:

  • اكتشاف الأخطاء أثناء التطوير
  • توثيق ذاتي للكود
  • أداء أفضل في بعض الحالات
  • أكثر ملاءمة للمشاريع الكبيرة

Dynamic Typing (JavaScript, Python, Ruby)

مثال في JavaScript

JavaScript 
// يمكن تغيير النوع بسهولة
let age = 25;
age = "خمسة وعشرون"; // لا يوجد خطأ

// الدوال لا تتطلب أنواعاً محددة
function greet(name) {
    return "مرحباً " + name;
}

greet(42); // يعمل ولكن قد يكون غير متوقع

مثال في Python

Python 
# Python مثال على
age = 25
age = "خمسة وعشرون"  # لا يوجد خطأ

def greet(name):
    return "مرحباً " + name

# سيعطي خطأ فقط عند التشغيل إذا تم تمرير عدد
print(greet("أحمد"))

مميزات Dynamic Typing:

  • مرونة أكبر في كتابة الكود
  • سهولة التعلم للمبتدئين
  • تطوير أسرع للنماذج الأولية
  • لا يحتاج إلى ترجمة (Compilation)

أيهما أفضل؟

لا يوجد إجابة واحدة، يعتمد على احتياجات المشروع. المشاريع الكبيرة والمعقدة تستفيد أكثر من Static Typing، بينما النماذج الأولية السريعة أو البرامج الصغيرة قد تكون أكثر ملاءمة لـ Dynamic Typing.

ملف تهيئة TypeScript

إنشاء الملف

strict

تفعيل جميع خيارات الصرامة (يفضل تفعيله)

outDir

المجلد الذي سيتم حفظ الملفات المترجمة فيه

strictNullChecks

يتحقق من القيم null و undefined

noImplicitReturns

noUnusedLocals

يكشف عن المتغيرات غير المستخدمة

خيارات المخرجات

sourceMap

ينشئ ملفات source map لتصحيح الأخطاء

removeComments

يزيل التعليقات من الملفات المترجمة

declaration

ينشئ ملفات .d.ts للأنواع

outFile

يجمع كل الملفات في ملف واحد

ملف tsconfig.json كامل

{
  "compilerOptions": {
    "target": "ES6",
    "module": "commonjs",
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true,
    "noImplicitReturns": true,
    "noUnusedLocals": true,
    "esModuleInterop": true,

نصائح مهمة

  • يفضل دائماً تفعيل الخيار strict لضمان أفضل ممارسات TypeScript
  • استخدم outDir و rootDir لتنظيم ملفات المشروع
  • noImplicitAny يساعد في تجنب استخدام any بشكل غير مقصود
  • sourceMap مفيد لتصحيح الأخطاء في بيئة التطوير

Type Annotation

الأنواع الأساسية والمتقدمة في TypeScript

الأنواع الأساسية

string

للقيم النصية

let name: string = "أحمد";
number

لجميع الأرقام

let age: number = 25;
boolean

للقيم المنطقية

let isActive: boolean = true;

المشاكل والحلول

المشكلة في JavaScript

function getLength(input) {
    return input.length;
}

// يعمل مع النصوص
getLength("hello"); // 5

// ولكن مع الأرقام سيعطي undefined
getLength(123); // undefined

الحل في TypeScript

function getLength(input: string): number {
    return input.length;
}

// يعمل فقط مع النصوص
getLength("hello"); // 5

// خطأ أثناء التطوير
getLength(123); // Error

الأنواع المركبة

Union Types

تسمح بتعريف متغير يمكن أن يكون من أكثر من نوع

// يمكن أن يكون string أو number
let id: string | number;
id = "ABC123"; // صحيح
id = 123; // صحيح
id = true; // خطأ

// مع المصفوفات
let mixedArray: (string | number)[] = [1, "two", 3];

مصفوفات متعددة الأبعاد

كيفية تعريف مصفوفات تحتوي على مصفوفات

// مصفوفة من مصفوفات الأعداد
let matrix: number[][] = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

// مصفوفة مختلطة
let mixedMatrix: (number | string)[][] = [
    [1, "two", 3],
    ["four", 5, "six"]
];

الدوال في TypeScript

أنواع المعاملات، القيم المرجعة، والخيارات المتقدمة

أساسيات الدوال

تحديد أنواع المعاملات

// دالة بمعاملات محددة الأنواع
function add(a: number, b: number): number {
    return a + b;
}

// الاستدعاء الصحيح
add(5, 10); // 15

// خطأ: المعامل الثاني يجب أن يكون عدد
add(5, "10"); // Error

noImplicitReturns

تأكد من أن جميع مسارات الدالة ترجع قيمة

// مع noImplicitReturns: true
function checkAge(age: number): string {
    if (age >= 18) {
        return "مسموح";
    }
    // Error: ليس كل مسار يرجع قيمة
}

خيارات متقدمة للدوال

المعاملات الاختيارية

// المعامل الاختياري باستخدام ?
function greet(name: string, title?: string): string {
    return title ? `مرحباً ${title} ${name}` 
               : `مرحباً ${name}`;
}

greet("أحمد"); // "مرحباً أحمد"
greet("أحمد", "الدكتور"); // "مرحباً الدكتور أحمد"

المعاملات الافتراضية

// قيمة افتراضية للمعامل
function createUser(
    name: string,
    isAdmin: boolean = false
): { name: string, isAdmin: boolean } {
    return { name, isAdmin };
}

// الاستدعاء بدون قيمة isAdmin
createUser("أحمد"); // { name: "أحمد", isAdmin: false }

Rest Parameters

// جمع جميع المعاملات في مصفوفة
function sum(...numbers: number[]): number {
    return numbers.reduce((a, b) => a + b, 0);
}

sum(1, 2, 3); // 6
sum(1, 2, 3, 4, 5); // 15
// مع أنواع مختلفة
function printInfo(...info: [string, number, boolean]): void {
    console.log(`الاسم: ${info[0]}, العمر: ${info[1]}, نشط: ${info[2]}`);
}

printInfo("أحمد", 25, true);

void و never

void

عندما لا ترجع الدالة أي قيمة

function logMessage(message: string): void {
    console.log(message);
    // لا يوجد return
}

never

عندما لا تنتهي الدالة أبداً

function throwError(message: string): never {
    throw new Error(message);
}

function infiniteLoop(): never {
    while (true) {
        // حلقة لا نهائية
    }
}

Type Aliases و Tuples

إنشاء أنواع مخصصة وهياكل بيانات محددة

Type Aliases

تسمح بتعريف اسم لنوع معين أو مجموعة من الأنواع لاستخدامها لاحقاً.

تعريف Type Alias

// تعريف نوع مخصص
type User = {
    id: number;
    name: string;
    email: string;
    isActive: boolean;
};

// استخدام النوع المخصص
const user: User = {
    id: 1,
    name: "أحمد",
    email: "ahmed@example.com",
    isActive: true
};

Literal Types

// تحديد قيم محددة يمكن للمتغير أن يأخذها
type Status = "pending" | "completed" | "failed";

let currentStatus: Status = "pending";
currentStatus = "completed"; // صحيح
currentStatus = "error"; // خطأ

// مع الدوال
function setStatus(status: Status): void {
    console.log(`تم تغيير الحالة إلى ${status}`);
}

Tuples

تمثل مصفوفة بطول ثابت وأنواع محددة لكل عنصر.

تعريف Tuple

// tuple لمعلومات المستخدم
let userInfo: [string, number, boolean];
userInfo = ["أحمد", 25, true]; // صحيح
userInfo = [25, "أحمد", true]; // خطأ

// tuple لإحداثيات نقطة
type Point = [number, number];
const p1: Point = [10, 20];
const p2: Point = [30, 40];

readonly Tuples

// tuple للقراءة فقط
let userInfo: readonly [string, number] = ["أحمد", 25];

// يمكن قراءة العناصر
const name = userInfo[0]; // "أحمد"

// لا يمكن التعديل
userInfo[0] = "محمد"; // Error
userInfo.push("جديد"); // Error

// مع Type Alias
type ReadonlyUser = readonly [string, number];
const user: ReadonlyUser = ["أحمد", 25];

الفرق الكامل بين JavaScript و TypeScript

شرح عملي مع أمثلة كود مفصلة

1. الأنواع الأساسية (Primitive Types)

JavaScript

// لا يوجد تحديد للأنواع
let name = "أحمد";
let age = 25;
let isActive = true;

// المشكلة: يمكن تغيير النوع بسهولة
age = "خمسة وعشرون"; // لا يوجد خطأ

TypeScript

// تحديد الأنواع أثناء التعريف
let name: string = "أحمد";
let age: number = 25;
let isActive: boolean = true;

// الحماية: سيظهر خطأ أثناء التطوير
age = "خمسة وعشرون"; // Error: Type 'string' is not assignable to type 'number'

💡 ملاحظة تعليمية:

TypeScript يمنع تغيير نوع المتغير بعد تعريفه، مما يقلل الأخطاء في البرنامج.

2. الدوال (Functions)

JavaScript

                        

                        
function sum(a, b) {
    return a + b;
}

// مشاكل محتملة:
console.log(sum(5, "3")); // "53" بدلاً من 8
console.log(sum(10)); // NaN

TypeScript

function sum(a: number, b: number): number {
    return a + b;
}

// الحماية:
console.log(sum(5, "3")); // Error
console.log(sum(10)); // Error: Expected 2 arguments, but got 1

// حلول متقدمة:
function safeSum(a: number, b: number = 0): number {
    return a + b;
}

💡 فوائد TypeScript:

  • تحديد أنواع المعاملات
  • تحديد نوع الإرجاع
  • معاملات اختيارية أو ذات قيم افتراضية

3. الكائنات والأنواع المعقدة

JavaScript

const user = {
    name: "نورا",
    age: 22
};

// المشاكل:
user.age = "22"; // مقبول
user.email = "nora@example.com"; // مقبول
console.log(user.nme); // undefined (خطأ مطبعي)

TypeScript

interface User {
    name: string;
    age: number;
    email?: string; // اختياري
}

const user: User = {
    name: "نورا",
    age: 22
};

// الحماية:
user.age = "22"; // Error
user.nme = "Ali"; // Error: Property 'nme' does not exist
user.email = "nora@example.com"; // مقبول

✨ TypeScript Features:

يقدم TypeScript نظامًا متكاملًا لتعريف أشكال الكائنات باستخدام interface أو type.

4. المصفوفات (Arrays)

JavaScript

const mixedArray = [1, "text", true, {name: "obj"}];

// المشاكل:
mixedArray.push(null); // مقبول
const numbers = [1, 2, 3];
numbers.push("4"); // مقبول

TypeScript

// مصفوفة مختلطة (غير مستحسنة)
const mixedArray: (number | string)[] = [1, "text"];

// الممارسة الأفضل:

المشروع العملي

تطبيق مفاهيم TypeScript على مشروع حقيقي

متطلبات المشروع

نريد بناء نظام لإدارة المستخدمين مع التحقق من الصلاحيات. النظام يجب أن:

  • يخزن معلومات المستخدمين (اسم، بريد إلكتروني، عمر، صلاحيات)
  • يتحقق من صلاحيات المستخدم قبل تنفيذ العمليات
  • يسمح بإضافة مستخدمين جدد مع التحقق من صحة البيانات
  • يقدم تقريراً عن المستخدمين النشطين

الحل بـ JavaScript

// المشاكل المحتملة:

// 3. أخطاء قد تظهر فقط أثناء التشغيل

const users = [];

function addUser(user) {
    users.push(user);

function getActiveUsers() {
    return users.filter(user => user.isActive);
}

// يمكن إضافة مستخدم ببيانات غير صالحة
addUser({
    name: "أحمد",
    email: "بريد-غير-صالح",
    age: "خمسة وعشرون", // يجب أن يكون عدد
    isActive: "نعم" // يجب أن يكون boolean
});

المشروع العملي المتقدم

نظام إدارة مهام لفريق العمل

متطلبات المشروع

  • تخزين معلومات المستخدمين (ID، اسم، بريد إلكتروني، دور: admin أو member)
  • تخزين معلومات المهام (ID، عنوان، تاريخ إنجاز، حالة، المستخدم المسؤول)
  • منع إسناد مهمة لشخص غير موجود
  • السماح فقط للـ admin بحذف المهام
  • إصدار تقرير بالمهام المتأخرة والمهام المكتملة

الحل بـ JavaScript

// مشاكل محتملة:

    tasks.push(task);

الحل بـ TypeScript

// تعريف الأنواع
type Role = "admin" | "member";

type User = {
    id: number;
    name: string;
    email: string;
    role: Role;
};

type TaskStatus = "pending" | "done";
type Task = {
    id: number;
    title: string;
    deadline: Date;
    status: TaskStatus;
    assignedTo: number; // ID المستخدم
};

const users: User[] = [];
const tasks: Task[] = [];

function addUser(user: User): void {
    users.push(user);
}

function addTask(task: Task): void {
    const assignedUser = users.find(u => u.id === task.assignedTo);
    if (!assignedUser) {
        throw new Error("المستخدم غير موجود");
    }
    tasks.push(task);
}

function deleteTask(taskId: number, userId: number): void {
    const user = users.find(u => u.id === userId);
    if (!user || user.role !== "admin") {
        throw new Error("غير مسموح لك بحذف المهام");
    }
    const index = tasks.findIndex(t => t.id === taskId);
    if (index > -1) tasks.splice(index, 1);
}

addUser({ id: 1, name: "أحمد", email: "ahmed@example.com", role: "admin" });
addUser({ id: 2, name: "منى", email: "mona@example.com", role: "member" });

addTask({ id: 1, title: "تسليم المشروع", deadline: new Date("2025-08-20"), status: "pending", assignedTo: 2 });

كيف يحل TypeScript المشاكل؟

  • يمنع إدخال بيانات غير صحيحة للـ Users والـ Tasks
  • يضمن أن القيم مثل الدور (Role) أو الحالة (Status) تأتي من قائمة محددة
  • يتحقق من وجود المستخدم قبل إسناد المهمة
  • يفرض صلاحيات الحذف على الـ admin فقط
  • يوفر توثيق ذاتي لهيكل البيانات

  • استخدم Math.sqrt() وMath.pow() لحساب المسافة
