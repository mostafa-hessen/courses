الفرق بين block, inline, inline-block

النوع السلوك أمثلة يقبل width/height
block يأخذ العرض الكامل ويبدأ سطر جديد div, p, h1-h6, section
inline يتدفق مع النص ولا يبدأ سطر جديد span, a, strong, em
inline-block يجمع بين الاثنين (يتدفق مع النص ويقبل أبعاد) button, input, select

justify-content

تتحكم في توزيع العناصر على المحور الرئيسي (أفقي أو عمودي حسب flex-direction).

flex-start
center
flex-end
space-between
space-around
.container {
  justify-content: center; /* flex-start | flex-end | center | space-between | space-around | space-evenly */
}
  • flex-start: العناصر تبدأ من بداية الحاوية
  • flex-end: العناصر تصطف في نهاية الحاوية
  • center: العناصر في المنتصف
  • space-between: توزيع العناصر مع ترك مسافة متساوية بينها فقط
  • space-around: توزيع العناصر مع ترك مسافة متساوية حول كل عنصر
  • space-evenly: توزيع العناصر بمسافات متساوية تمامًا

align-items

تتحكم في محاذاة العناصر على المحور المتقاطع (العمودي إذا كان flex-direction: row).

flex-start
center
flex-end
.container {
  align-items: center; /* flex-start | flex-end | center | baseline | stretch */
}
  • flex-start: العناصر تصطف في أعلى الحاوية
  • flex-end: العناصر تصطف في أسفل الحاوية
  • center: العناصر في منتصف الحاوية
  • baseline: العناصر تصطف حسب خط الأساس للنص
  • stretch: العناصر تتمدد لملء الحاوية (افتراضي)

خصائص عناصر Flexbox (شرح مفصل)

الخصائص التي تطبق على العناصر الفرعية (Flex Items)

flex-grow

1
2 (flex-grow: 2)
1

flex-grow يحدد مدى قدرة العنصر على النمو لملء المساحة الفارغة داخل الحاوية.
كيف يعمل؟ إذا كان هناك مساحة متبقية بعد توزيع الأحجام الأساسية (flex-basis)، يتم توزيعها حسب قيمة flex-grow لكل عنصر.
مثال: إذا كان لدينا 3 عناصر بقيم (1, 2, 1) فالعنصر الثاني سيأخذ ضعف المساحة الإضافية مقارنة بالعنصر الأول أو الثالث.
القيمة الافتراضية: 0 (لا ينمو)

flex-shrink

1 (flex-shrink: 1)
0 (لا يتقلص)
1 (flex-shrink: 1)

flex-shrink يحدد مدى سماح العنصر بالتقلص عندما لا توجد مساحة كافية في الحاوية.
كيف يعمل؟ إذا كانت العناصر أكبر من الحاوية، يتم تقليصها حسب قيمة flex-shrink.
مثال: عنصر بقيمة 0 لن يتقلص أبدًا، بينما عنصر بقيمة 1 أو أكثر سيتقلص حسب النسبة.
القيمة الافتراضية: 1 (يتقلص)

flex-basis

100px
150px
200px

flex-basis يحدد الحجم الأساسي (الابتدائي) للعنصر قبل توزيع المساحة الزائدة أو الناقصة.
كيف يعمل؟ هو مثل width أو height (حسب اتجاه flex)، لكنه يأخذ الأولوية إذا تم تحديده مع flex.
مثال: إذا كان flex-basis: 150px، سيبدأ العنصر بهذا الحجم ثم ينمو أو يتقلص حسب flex-grow و flex-shrink.
القيمة الافتراضية: auto (بحسب المحتوى أو width/height)

align-self

افتراضي
align-self: center
align-self: flex-end

align-self يسمح لك بتغيير محاذاة عنصر معين على المحور المتقاطع (Cross Axis)، بغض النظر عن قيمة align-items في الحاوية.
كيف يعمل؟ إذا أردت أن يكون عنصر واحد فقط في المنتصف أو في الأسفل، استخدم align-self عليه فقط.
القيم: auto (افتراضي)، flex-start، flex-end، center، baseline، stretch.
مفيد لتخصيص محاذاة عنصر واحد فقط

/* أمثلة على خصائص العناصر */
.item {
  flex: 1 1 auto; /* اختصار لـ flex-grow, flex-shrink, flex-basis */
}

.special-item {
  flex-grow: 2; /* ينمو ضعف العناصر الأخرى */
  flex-shrink: 0; /* لا يتقلص أبداً */
  flex-basis: 200px; /* يبدأ من 200px */
  align-self: center; /* محاذاة في المنتصف رأسياً */
}

ملخص سريع

  • flex-grow: يتحكم في النمو عند وجود مساحة إضافية.
  • flex-shrink: يتحكم في التقلص عند ضيق المساحة.
  • flex-basis: يحدد الحجم الابتدائي للعنصر.
  • align-self: يخصص المحاذاة لعنصر واحد فقط.
  • flex: اختصار للخصائص الثلاثة معًا: flex: grow shrink basis;

الوحدات المرنة في CSS

وحدات القياس النسبية

  • %: نسبة مئوية من حجم العنصر الأب
  • em: نسبة إلى حجم خط العنصر الحالي (تتأثر بالوراثة)
  • rem: نسبة إلى حجم خط العنصر الجذر (html) - الأفضل للاستخدام
  • vw: 1% من عرض نافذة العرض (viewport width)
  • vh: 1% من ارتفاع نافذة العرض (viewport height)

هذا عنصر باستخدام em (1em = 16px افتراضيًا)

هذا العنصر الداخلي (1.5em = 24px)

هذا عنصر باستخدام rem (1rem = 16px افتراضيًا)

هذا العنصر الداخلي (1.5rem = 24px)
/* في عنصر الجذر (root) */
:root {
  --font-size-root: 16px; /* تحديد حجم الخط الأساسي */
}

body {
  font-size: var(--font-size-root); /* استخدام rem يعتمد على هذا الحجم */
}

نقاط التوقف (Breakpoints) وأحجام الشاشات

نوع الجهاز عرض الشاشة نقطة التوقف الشائعة مثال استعلام
هواتف صغيرة أقل من 576px لا يحتاج نقطة توقف -
هواتف كبيرة 576px - 767px 576px @media (min-width: 576px)
أجهزة لوحية 768px - 991px 768px @media (min-width: 768px)

استعلامات الوسائط (Media Queries)

استعلامات الوسائط تسمح لك بتطبيق أنماط CSS مختلفة بناءً على خصائص الجهاز (مثل عرض الشاشة).

الشعار
الرئيسية من نحن الخدمات اتصل بنا
/* شريط تنقل متجاوب */
.navbar {
  display: flex;
  flex-direction: column;
}

.nav-links {
  display: flex;
  flex-direction: column;
  gap: 10px;
}

/* نقطة توقف للأجهزة اللوحية */
@media (min-width: 768px) {
  .navbar {
    flex-direction: row;
    justify-content: space-between;
  }
  
  .nav-links {
    flex-direction: row;
  }
}

/* نقطة توقف لأجهزة الكمبيوتر */
@media (min-width: 992px) {
  .navbar {
    padding: 20px;
  }
}

التحريك والانتقالات في CSS

الفرق بين الانتقالات (Transitions) والتحريك (Animations)

الانتقالات (Transitions)

تتحكم في كيفية الانتقال بين حالتين (مثل hover)

جربني
.element {
  transition: all 0.3s ease;
}

.element:hover {
  transform: scale(1.2);
  background: #13b9fd;
}

التحريك (Animations)

تسمح بتحريك متعدد الخطوات باستخدام @keyframes

متحرك
@keyframes slideAnimation {
  0% { transform: translateX(0); }
  50% { transform: translateX(100px); }
  100% { transform: translateX(0); }
}

.element {
  animation: slideAnimation 3s infinite;
}

خصائص التحريك (Animation Properties)

  • animation-name: اسم الـ keyframe
  • animation-duration: مدة التحريك (مثال: 2s)
  • animation-timing-function: سرعة التحريك (ease, linear, etc.)
  • animation-delay: تأخير قبل بدء التحريك
  • animation-iteration-count: عدد المرات (infinite, 3, etc.)
  • animation-direction: اتجاه التحريك (normal, reverse, alternate)
  • animation-fill-mode: حالة العنصر بعد الانتهاء
  • animation-play-state: التحكم بتشغيل/إيقاف التحريك

الخاصية المختصرة animation

/* الصيغة المختصرة: */
animation: name duration timing-function delay iteration-count direction fill-mode play-state;

/* مثال عملي: */
.element {
  animation: slideAnimation 3s ease-in-out 0.5s infinite alternate both running;
}
  • name: اسم الـ keyframes (إجباري)
  • duration: مدة التحريك (إجباري)
  • timing-function: (اختياري)
  • delay: (اختياري)
  • iteration-count: (اختياري)
  • direction: (اختياري)
  • fill-mode: (اختياري)
  • play-state: (اختياري)

تحديد المواقع في CSS

أنواع تحديد المواقع واستخداماتها

Static (ثابت)

الوضع الافتراضي - يتبع التدفق الطبيعي للصفحة

.element {
  position: static; /* الافتراضي */
}

Relative (نسبي)

يُحرّك بالنسبة لموقعه الأصلي - لا يؤثر على العناصر الأخرى

.element {
  position: relative;
  top: 20px;
  left: 30px;
}

Absolute (مطلق)

يُحدّد بالنسبة لأقرب عنصر أب له موضع محدد (غير static)

.parent {
  position: relative; /* نقطة مرجعية */
}
.child {
  position: absolute;
  top: 0;
  right: 0;
}

Fixed (ثابت على الشاشة)

ثابت بالنسبة لشاشة العرض - يبقى في مكانه عند التمرير

استخدام شائع: شريط تنقل ثابت في أعلى الصفحة

.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 1000;
}

Sticky (لاصق)

يتصرف كـ relative حتى يصل إلى نقطة محددة، ثم يثبت كـ fixed

استخدام شائع: عناوين الأقسام في القوائم الطويلة

.section-title {
  position: sticky;
  top: 20px; /* عندما يصل إلى 20px من الأعلى يثبت */
}

z-index وترتيب الطبقات

كيفية التحكم في ترتيب العناصر المتداخلة

z-index: 2
z-index: 3
<!-- مثال HTML -->

  <div class="box box3">الطبقة 3</div>

/* مثال CSS */
.container {
  position: relative;
  height: 200px;
}

.box {
  position: absolute;
  width: 120px;
  height: 120px;

}

.box3 {
  top: 70px;
  left: 70px;
  z-index: 3;
  background: rgba(229, 62, 62, 0.7);
}

ما هو z-index؟

يحدد ترتيب العناصر المتداخلة على المحور Z (عمق الصفحة). كلما زاد الرقم، كلما ظهر العنصر أقرب إلى المستخدم.

سياق الترتيب (Stacking Context)

يتم إنشاؤه عندما يتم تحديد موضع لعنصر واستخدام z-index. العناصر الفرعية تكون ضمن هذا السياق.

خطأ شائع

z-index يعمل فقط على العناصر التي تم تحديد موقعها (relative, absolute, fixed, sticky). ليس له تأثير على العناصر الافتراضية (static).
